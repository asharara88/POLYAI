# Events

Owned by `events`. One folder per active event. Each folder has:
- `plan.md` — the event plan (audience, headcount, vendors, budget, run-of-show, communications calendar)
- `invitation-list.md` (optional) — attendees + status
- `rsvp-log.md` (optional) — RSVPs received with notes
- `run-of-show.md` (optional) — full minute-by-minute on the day
- `debrief.md` (post-event) — actual vs. planned, lessons-learned

Event types follow the conventions in the `events` agent prompt: pre-launch private previews, public launch days, broker tier-1 private events, broker tier-2/3 group sessions, wealth-channel previews, VVIP / royal-attended openings, owner / loyalty events, industry events, internal / partner events.
