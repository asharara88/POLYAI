# Playbooks

Reusable plays. Each playbook captures a motion that's worked (or, when archived, one that didn't and why).

Suggested files to add as you populate the system:

- `lead-scoring.md` — how `inbound-qualifier` scores fit and intent
- `sales-plays.md` — discovery and demo plays for `account-executive`
- `pricing.md` — pricing library used by `proposal`
- `proposals/` — proposal and SOW templates
- `battle-cards/` — one file per named competitor, owned by `competitive-intel`
- `compliance/` — region- and industry-specific rules used by `compliance`
- `localization/` — per-market language and cultural adaptation notes used by `localization`
- `archive/` — failed plays kept for reference

Discipline: keep each playbook short and falsifiable. A 30-page sales-play doc gets read by no one.
